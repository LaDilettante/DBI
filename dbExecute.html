Execute an SQL statement that does not produce a result set

Usage

dbExecute(conn, statement, ...)

Arguments

conn
A DBIConnection object, as produced by dbConnect.
statement
a character vector of length 1 containing SQL.
...
Other parameters passed on to methods.

Value

The number of rows affected by the statement

Description

This function should be used when you want to execute a non-SELECT query on table (ex: UPDATE, DELETE, INSERT INTO, DROP TABLE, ...). It will execute the query and return the number of rows affected by the operation. The default implementation calls dbSendQuery and dbGetRowsAffected, which is useful if a backend does not distinguish between SELECT and non-SELECT queries.

Examples

con <- dbConnect(RSQLite::SQLite(), ":memory:")

dbWriteTable(con, "cars", head(cars, 3))



[1] TRUE



dbReadTable(con, "cars")   # there are 3 rows



  speed dist
1     4    2
2     4   10
3     7    4



dbExecute(con, "INSERT INTO cars (speed, dist)
                VALUES (1, 1), (2, 2), (3, 3);")



[1] 3



dbReadTable(con, "cars")   # there are now 6 rows



  speed dist
1     4    2
2     4   10
3     7    4
4     1    1
5     2    2
6     3    3




dbDisconnect(con)



[1] TRUE
